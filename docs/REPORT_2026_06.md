## 一、执行摘要

EnerGraph 是公司青山大模型 V3.0 五层架构中 **第 3 层（决策层）** 的核心代码实现，定位为整个能碳管理平台的"AI 编排大脑"。项目自 5 月中旬启动开发，经过一个月的高强度迭代，**已完成 7 个主要功能阶段**，系统从原型验证阶段正式进入**企业级交付就绪状态**。

**当前关键成果**：

- ✅ **首个业务身份 PowerAI（储能调度智能体）已具备完整的数据查询与解读能力**，后续可快速复制出能效诊断、碳管理、微电网等多个智能体——它们共享同一套决策层代码，只是挂载不同的业务模块
- ✅ **已全面打通福加能碳管理平台的 11 个真实运营数据接口**，AI 可直接查询 COP、能耗、报警、光伏、碳排、设备排名等实时数据并生成专业解读报告
- ✅ **完成企业级交付基础设施**：标准化 API 接口（含跨域支持和安全鉴权）、流式响应、自动化测试体系，已具备嵌入福加官方网页的条件
- ✅ **完成多智能体解耦架构升级**：支持多名开发者并行开发不同智能体，互不干扰，为团队扩张铺平道路

> **一句话概括**：决策层 Agent 已经"能听、能查、能说、能跳"，下一阶段重点是接入算法层的预测与优化模型集群（共 9 大模型），使 PowerAI 从"实时数据查询解读助手"升级为"预测 + 决策"双能力的储能调度管家。

## 二、已完成的核心业务能力

### 2.1 福加运营数据全面打通

AI 智能助手已与公司现有的能碳管理平台 Java 后端 **深度集成**，目前可实时查询以下运营数据：

| 数据类别 | 能力说明 | 业务价值 |
|----------|---------|---------|
| **机房能效（COP）** | 查询机房整体 COP、单台机组 COP、蒸发器/冷凝器温度、实时功率 | 运营人员无需手动翻查多个页面，一句话即可获得完整能效画像 |
| **能耗汇总** | 总用电量、光伏发电、储能充放电、电网交互、碳减排、峰值/平均负荷 | 管理层可快速获取当日/历史能耗全貌，含光储分项明细 |
| **活跃报警** | 当前报警级别、设备、报警信息、时间 | 第一时间掌握异常，缩短故障响应时间 |
| **碳排放信息** | 月度光伏发电量、碳减排量、累计碳减排、环比数据 | 为 ESG 报告和碳资产管理提供数据支撑 |
| **光伏发电** | 月度发电量与收益趋势、单日发电量与峰值功率 | 光伏投资回报可视化，辅助运维决策 |
| **设备用电排名** | 全厂 Top5 设备用电量、机房设备能耗占比 | 快速定位高耗能设备，辅助节能改造优先级 |
| **室外环境参数** | 温度、湿度、湿球温度、焓值 | 辅助判断制冷需求与环境关联 |
| **能效日历** | 日度/月度 COP、制冷量、用电量、电价趋势 | 能效趋势追踪，识别异常波动 |
| **通用能效查询** | 8 种机房级能效参数的灵活查询 | 覆盖更广泛的运营分析场景 |

**安全认证创新**：针对福加后端的 Token 过期机制，自研了**全自动 Token 刷新系统**——模拟前端加密登录流程，实现 7×24 无人值守运行，无需人工干预密钥更新。

### 2.2 高阶多意图识别（Phase 7）

**核心能力**：用户一句话中包含多个独立请求时，AI 能自动识别、拆分、并行执行。

**示例场景**：
> 用户输入：*"帮我查一下今天的 COP，顺便看看有没有报警，再查下这个月的光伏发电量"*

**系统行为**：
1. 自动识别出 3 个独立意图
2. 判断三者之间无数据依赖 → 并行调用 3 个数据接口
3. 生成**分段报告**（## 1. COP 查询 / ## 2. 报警情况 / ## 3. 光伏发电），附综合小结

**业务价值**：大幅提升运营人员的查询效率，减少反复提问的交互成本。

### 2.3 专业级 HVAC 知识问答（Phase 3）

构建了涵盖 **5,605 条**暖通空调专业语料的知识库，覆盖：
- 国内外暖通设计规范（GB、ASHRAE 等）
- 能效计算方法与公式
- 故障诊断与排查指南
- 地铁站 / 商业建筑专项场景
- 节能优化策略

**防幻觉机制**（行业领先实践）：
- **置信度拒答**：当检索结果与问题相关度不足时，AI 明确告知"知识库中暂无相关信息"，**绝不编造答案**
- **来源引用**：每条回答末尾标注引用来源（≤3 条），可溯源可验证
- **去重过滤**：自动剔除高度重复的检索片段，确保回答信息密度

### 2.4 前端集成与页面联动

AI 智能助手已完成与**福加官方前端网页**的对接准备：

- **流式输出**：回答逐字呈现（首字延迟 < 2 秒），用户体验流畅
- **智能页面跳转**：AI 回答能耗数据时，底部自动附带"跳转到能耗分析页面"的链接；回答报警信息时自动附带报警列表页面链接——覆盖 24 个可访问页面和 10 个受限页面
- **思考过程透明**：实时展示 AI 的意图识别和工具调用过程，完成后可折叠收起，兼顾专业用户和普通用户的体验需求
- **标准化接口文档**：已提供完整的 Vue.js 代码示例和 TypeScript 类型定义，前端团队可快速嵌入

## 三、底层架构与工程基建突破

### 3.1 多智能体解耦架构（6 月重大重构成果）

完成了从"单体架构"到"多智能体 Subgraph 架构"的全面升级，这是面向团队扩张的关键基建：

**重构前**：所有业务逻辑集中在少数文件中，多人同时修改必然产生代码冲突。

**重构后**：

| 维度 | 效果 |
|------|------|
| **目录隔离** | 每个智能体独占独立文件夹，开发者 A 改 PowerAI、开发者 B 改碳管理，文件层面零冲突 |
| **配置隔离** | 每个智能体有独立的 Prompt 配置文件，修改互不影响 |
| **即插即用** | 新增智能体只需"创建文件夹 + 注册一行代码"，主系统无需改动 |
| **测试隔离** | 各智能体独立运行自己的测试套件，快速验证不影响全局 |

**战略意义**：公司规划的 PowerAI 储能调度、能效诊断、碳管理、微电网、充电调度等多个智能体，未来都可以由不同开发者**并行推进**，不再互相阻塞。

### 3.2 高性能流式响应体系

| 技术特性 | 用户感知 |
|---------|---------|
| **SSE 流式推送** | 回答像"打字"一样逐字出现，无需等待完整生成 |
| **5 种细粒度事件类型** | 前端可分别控制"思考过程"、"工具调用"、"最终回答"的展示方式 |
| **异步非阻塞架构** | 多个用户同时提问不会互相卡顿 |
| **跨域 + 鉴权** | 安全嵌入福加官方网页，防止未授权访问 |

### 3.3 工业级开发与质量保障

| 指标 | 现状 |
|------|------|
| **自动化测试** | 60+ 项测试用例全部通过，覆盖核心逻辑、数据质量、接口集成 |
| **代码规范** | 统一的文件头标注、函数注释、类型约束，代码可读性强 |
| **Git 协作流** | 分支保护 + 代码审查模板 + 提交规范，保障多人协作质量 |

| 制冷系统寻优 | ✅ 接口契约已定义 | MCP 协议（热插拔） |
| 储能调度优化 | ✅ 接口契约已定义 | MCP 协议（热插拔） |

**对接方式**：算法工程师只需按已定义的接口规范（输入字段、输出字段、错误码）封装模型为 MCP Server，Agent 侧即可**无需修改主代码**直接调用。完整的接口契约文档（`MCP_INTERFACE_SPEC.md`）已准备就绪，含详细的 JSON 请求/响应示例。

**Agent 智能调度双路径设计**：
- **预测查询路径**：用户询问"明天预计用多少电"、"光伏发电趋势"、"电价走势"等单一预测问题时，Agent 直接调用对应预测模型，秒级返回预测结果
- **综合调度路径**：用户要求"出个充放电策略"时，Agent 调用综合调度优化模型，由优化模型内部联动负荷/光伏/电价等预测引擎，综合多维因素后返回 2-3 个候选方案（稳健型 / 均衡型 / 激进型）

两条路径按用户意图智能选择，Agent 具备全链路算法模型的调度能力。

**当前状态**：Agent 侧的接口适配层和降级逻辑已全部就绪，当前基于实时运营数据独立运行。算法模型接入后，系统将在现有数据查询能力的基础上，叠加前瞻性的预测与调度决策能力。

## 六、总结

EnerGraph 决策层项目在一个月内完成了从 0 到 1 的企业级基础设施建设：

- **业务侧**：已能查询福加厂区全部核心运营数据并生成专业解读，具备嵌入官方网页交付使用的条件
- **架构侧**：多智能体解耦架构为团队扩张和多产品线并行开发铺平道路
- **生态侧**：标准化 MCP 接口契约使算法模型可热插拔接入，两个团队可独立并行推进
- **质量侧**：自动化测试 + 代码规范 + 文档体系三位一体，保障长期可维护性

**下一阶段核心目标**：在现有数据查询与解读能力的基础上，接入算法层的预测与优化模型集群，使 PowerAI 智能体升级为兼具单预测查询和综合调度决策能力的储能调度管家。
