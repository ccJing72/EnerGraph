# EnerGraph PRD — 青山大模型决策层产品需求文档

> 产品视角的系统功能定义。开发新功能前先读本文，确认"做什么"和"不做什么"。
> 版本: v1.0 | 最后更新: 2026-06-12

---

## 1. 产品定位

**一句话**：青山大模型决策层——福加能碳管理平台的 AI 编排大脑，自己不做数值计算，通过 MCP 协议调用算法模型 + REST API 查询运营数据，将结果转化为用户可理解的专业报告。

**在 V3.0 五层架构中的位置**：第 3 层（决策层）。上接第 5 层应用层（前端交互），下接第 2 层算法模型层（计算引擎）。

**第一个业务身份**：PowerAI 储能调度智能体（光伏预测 + 负荷预测 → 综合决策 → 充放电策略）。未来通过不同 Skill 配置可扩展为能效诊断、碳管理等其他智能体。

## 3. 用户场景（Use Cases）

### UC-1: HVAC 专家问答 ✅ 已实现

**触发**：用户提出暖通空调专业问题（规范查询、能效计算、故障诊断、节能优化）

**期望行为**：
- 从 5605 条 HVAC 知识库中 RAG 检索
- 给出准确、引用规范条文的专业回答
- 置信度低时明确拒答，不编造

**边界**：
- 知识库无相关内容 → 拒答（"知识库中暂无相关信息"）
- 不得用自身知识猜测性回答
- 引用来源标注（≤3 条）

---

### UC-2: 福加运营数据查询 ✅ 已实现

**触发**：用户询问 COP / 能耗 / 报警 / 碳排 / 光伏 / 设备排名 / 环境参数等

**期望行为**：
- 调用对应的福加 Java 后端 REST API
- 数据解读 + Markdown 格式化呈现
- 底部自动附加相关页面跳转链接

**已有工具**（10 个真实 API）：

| 工具 | 数据内容 |
|------|---------|
| fetch_cop_data | 机房 COP + 机组 COP + 运行参数 |
| fetch_energy_summary | 总用电 / 光伏 / 储能 / 碳减排 |
| fetch_active_alarms | 当前活跃报警列表 |
| fetch_carbon_info | 碳排 / 光伏月累计 |
| fetch_photovoltaic_monthly | 光伏月度发电量 + 收益 |
| fetch_photovoltaic_daily | 单日光伏发电量 + 峰值功率 |
| fetch_energy_usage | 全厂用电量（今日/本月） |
| fetch_device_rank | 设备用电排名（全厂/机房） |
| fetch_environment_params | 室外温湿度 / 焓值 |
| fetch_efficiency_calendar | 能效日历（日度/月度） |
| fetch_efficiency_detail | 通用能效参数查询（8 种） |

### UC-3: 多意图识别 ✅ 已实现

**触发**：单次输入包含多个独立请求（如"查 COP，顺便看看报警"）

**期望行为**：
- cognitive_parser 自动识别所有意图
- 无依赖的意图并行执行
- 有依赖的意图串行执行
- interpreter 生成分段报告 + 综合小结

### UC-5: 储能调度分析（PowerAI 核心）⏳ 待实现

**触发**：用户需要储能充放电策略 / 负荷预测 / 光伏预测 / 综合调度建议

**期望行为**（算法模型 MCP 接入后）：
1. 调用光伏预测模型 → 未来 24-72h 出力预测
2. 调用负荷预测模型 → 未来 24-72h 用电预测
3. 调用电价预测模型 → 峰谷电价趋势
4. 调用储能调度优化模型 → 生成 2-3 个候选方案（稳健/均衡/激进）
5. 解读方案差异、收益预估、风险提示
6. 用户选择方案后执行并持续监控

**当前状态**：算法模型尚未就绪，MCP 接口待对齐。

## 4. 非功能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 响应时间 | 首 token < 2s（SSE 流式） | P99 延迟控制在 2s 以内 |
| 数据真实性 | 禁止编造运营数据 | 所有数据必须来自工具调用 |
| 拒答机制 | RAG low_confidence → 拒答 | distance > 阈值时不猜测 |
| 多 LLM 支持 | DeepSeek V4 / OpenAI / Claude | `LLM_PROVIDER` 一键切换 |
| 安全 | Prompt Injection 防护 | 未来接入 NeMo Guardrails |
| 可观测性 | LangSmith 全链路追踪 | `LANGCHAIN_TRACING_V2=true` |

## 5. 禁止行为（红线）

1. **禁止**自行计算能源数据（COP、电量、碳排等），所有数据必须通过工具获取
2. **禁止**编造不存在的 API 数据或虚构运营指标
3. **禁止**在回答中引导页面跳转（"需要我跳转到XXX吗"）
4. **禁止**在 RAG 低置信度时用自己的知识猜测性回答 HVAC 问题
5. **禁止**将用户敏感数据（站点参数、设备 ID）泄露到外部
6. **禁止**在 Prompt 中硬编码（所有 Prompt 集中在 `prompts.yaml`）

## 7. 用户偏好学习（PowerAI 特色）

PowerAI 的核心差异化能力是"越用越聪明"：

- **方案选择偏好**：用户反复选择"稳健型" → Agent 自动调高安全权重
- **安全阈值偏好**：用户说"需量不能超过 1900kW" → 锁定为硬约束
- **反馈驱动调参**：用户说"光伏预测不准" → 自动触发模型微调

> 技术实现：贝叶斯偏好推断 + Thompson Sampling，20-30 次交互后收敛。
